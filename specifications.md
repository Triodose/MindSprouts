# MindSprout - 功能規格書 (Merged Functional Specifications)

本規格書以 **MindSprout (思芽)** 已開發的功能與架構為基礎，並融合 **XMind** 的核心功能規格（如結構混搭、大綱模式、標記多媒體、進階檔案相容性），整理出一份兼具「現有實作規格（V1.0）」與「未來藍圖規劃（V2.0）」的完整產品功能規格書。

---

## 1. 系統架構與基礎引擎 (V1.0 已實作)

### 1.1 排版渲染機制
*   **雙層混合渲染引擎**：使用標準 HTML DOM 搭配 CSS Flexbox 進行樹狀分支的層級排版，讓節點在新增、編輯、字型放大時自動流暢推開鄰近節點，100% 避免節點重疊；同時疊加 SVG 層，在背景繪製平滑的**三次貝氏曲線 (Cubic Bezier Curves)** 進行節點連結。
*   **虛擬根節點架構 (`virtual-root`)**：在底層 JSON 資料結構中，以隱藏的 `virtual-root` 作為最上層容器，其第一個子節點為「主根節點 (Root Topic)」，其他子節點則為獨立的「自由主題 (Floating Topics)」。此設計在完全相容單一 JSON 樹狀物件的同時，完美實現了單一畫布上存在多個獨立主體的需求。

### 1.2 畫布控制 (Zoom & Pan)
*   **平移畫布 (Pan)**：按住畫布空白處拖曳，可平移視角，畫布隨位移無限延伸。
*   **以游標為中心縮放 (Zoom-to-Cursor)**：滑鼠滾輪/觸控板縮放，縮放中心會自動黏著於游標位置（縮放範圍限制為 `0.3x` 至 `2.5x`），公式依據視口坐標變換防止偏移。
*   **雙擊新增自由節點**：雙擊畫布空白處，系統會自動將螢幕座標轉換為畫布內部的絕對座標，並在此座標下建立獨立的「自由主題」節點，且自動開啟文字編輯。

## 2. 核心功能模組

### 2.1 心智圖繪製與多結構混搭 (V2.0 規劃)
XMind 提供了 9 種核心思維結構，並允許在**同一個畫布中的不同分支**混搭使用不同的結構。MindSprout 目前 V1.0 預設採用「邏輯圖 (Logic Chart - 右側發散)」，未來將引進多結構混搭功能。

*   **經典心智圖 (Mind Map)**：從中心向四周發散的放射狀結構。
*   **邏輯圖 (Logic Chart)**：左至右或右至左的單向思維導向圖 (MindSprout V1.0 預設)。
*   **組織結構圖 (Org Chart)**：由上至下的階層樹狀結構，適用於組織架構或檔案目錄。
*   **樹狀圖 (Tree Chart)**：垂直排列的樹狀分支結構。
*   **括號圖 (Brace Map)**：由大括號組成的層次劃分圖，適合展示整體與部分的關係。
*   **時間軸 (Timeline)**：水平或垂直的順序事件軸，用於專案里程碑與歷史事件。
*   **魚骨圖 (Fishbone Diagram)**：骨架結構，專門用於因果分析與問題根源追蹤。
*   **分支結構混搭 (Mix & Match)**：使用者可以選取任一分支（Branch/Topic），並在其 Inspector 面板中將該分支單獨設定為不同於主圖的結構（例如：主圖為右側邏輯圖，但某一子分支採用魚骨圖進行分析，另一子分支採用組織結構圖）。

### 2.2 大綱模式 (Outliner - V2.0 規劃)
*   **一鍵切換**：提供心智圖與線性大綱的即時雙向切換。
*   **階層管理**：以直觀的條列清單（List）展示結構，便於文字創作者組織段落。
*   **同步編輯**：在大綱模式下新增、刪除或拖曳調整層級，心智圖結構會同步更新。

### 2.3 左側專案管理欄 (MapListSidebar - V1.0 已實作)
*   **折疊切換**：點擊左側收折按鈕可將專案欄收折至側邊，釋放畫布。
*   **建立心智圖**：點擊「建立新心智圖」會生成包含預設主節點的新檔案。
*   **原位重命名**：滑鼠懸停於列表項目上會出現「編輯筆圖示」，點擊可直接在列表輸入框內變更專案標題。
*   **刪除心智圖**：滑鼠懸停於列表項目上點擊「垃圾桶圖示」，經確認後刪除（防呆機制限制：至少必須保留一個檔案）。

### 2.4 頂部工具列 (Toolbar - V1.0 已實作)
*   **節點工具**：新增子節點、新增兄弟節點、刪除選取節點。
*   **歷史工具**：復原 (Undo)、重做 (Redo)（當前操作有變更時方可點擊）。
*   **畫布工具**：放大、縮小、置中畫布。
*   **快速鍵幫助**：點擊打開彈出式快速鍵操作指南對話框。

## 3. 節點操作與快速鍵規範 (V1.0 已實作)

當使用者選取節點且*非*文字編輯狀態時，觸發以下全域快速鍵：

| 快速鍵 | 觸發功能 | 備註說明 |
| :--- | :--- | :--- |
| **`Tab`** | 新增子節點 (Child Node) | 自動在右側長出並自動取得焦點，直接開啟文字編輯 |
| **`Enter`** | 新增兄弟節點 (Sibling Node) | 在同層級下方建立節點，若為 Root 則自動建立為 Floating 主題 |
| **`Space` / 雙擊** | 編輯節點文字 | 切換為 Input 輸入框，失去焦點 (Blur) 或按 Enter 自動存檔 |
| **`Delete` / `Backspace`** | 刪除選取節點 | 根節點 `virtual-root` 與視覺主根 `root` 無法被刪除 |
| **`Escape`** | 取消選取 / 關閉編輯 | 退出編輯模式（還原未確認文字）或清除選取狀態 |
| **`Ctrl + Z`** | 復原 (Undo) | 退回上一次節點結構或樣式的狀態 |
| **`Ctrl + Y`** | 重做 (Redo) | 前進至下一個變更狀態 |
| **`↑` / `↓` / `←` / `→`** | 移動選取目標 | 依據節點的父子或兄弟階層關係，上下左右跳躍選取 |

## 4. 進階拖曳與節點歸併 (Drag & Drop - V1.0 已實作)

### 4.1 分支拖曳 (Offset Dragging)
*   滑鼠左鍵點選節點卡片並拖曳時，該節點的屬性會被賦予一個相對於 Flex 預設排版的 `offset: {x, y}`。
*   **分支聯動**：在 DOM 結構中，子分支節點容器是父節點的子代，因此**拖曳父節點會帶動整個分支（包含子節點）整體平移**，維持樹狀版面完整。
*   **比例自適應**：拖曳移動量會自動依當前 Zoom 係數換算：`實際位移 = 螢幕位移 / transform.zoom`，使滑鼠指標與卡片中心完美同步，無漂移感。

### 4.2 拖曳歸併與碰撞偵測 (Drag-to-Reparent)
*   **滑鼠事件穿透 (Pointer-Events Penetration)**：拖曳開始時，該拖曳分支會被暫時賦予 CSS `pointer-events: none`。藉此，當呼叫 `document.elementFromPoint(x, y)` 進行碰撞偵測時，瀏覽器會直接穿透拖拉中的節點，取得底下的目標卡片，解決了拖拉時滑鼠永遠指著「自己」而無法偵測下方目標的問題。
*   **防錯保護**：拖曳至目標節點時，系統會比對 ID，**拒絕將節點歸併至它自己或它的任何子樹下**。
*   **視覺反饋**：若滑鼠懸停於合法目標節點，該節點會動態加入 `.drop-hover` 類別，呈現 **「藍色虛線邊框 + 外發光」** 的吸引特效。
*   **結構重組**：在目標節點上放開滑鼠時，節點會脫離原父節點，成為目標節點的子節點。此時**拖曳 Offset 自動歸零並清除**，讓節點自動對齊到新父節點的排版中。

## 5. 視覺自訂與多媒體標記 (V1.0 已實作 & V2.0 規劃)

### 5.1 樣式編輯面盤 (Inspector - V1.0 已實作)
*   **節點形狀**：圓角框 (Rounded)、線框 (Border)、下底線 (Underline)。
*   **文字尺寸與粗細**：支援字型大小（12px 至 22px）與字型粗細（300/500/700）微調。
*   **配色色塊**：提供 10 種精選配色色塊，可獨立變更節點的文字顏色、背景填滿色與邊框線條色。
*   **自適應文字顏色**：全域樣式整合進 CSS 變數，所有面板文字會隨主題深淺自動調節前景顏色，確保在 Midnight Slate 等深色主題下保持清晰的易讀性。
*   **收折按鈕防干擾**：展開/收折的 `"-"`/`"+"` 號圓點設定了 `onMouseDown` 冒泡攔截，點擊大圓點時會立即收折子節點，**完全不會誤觸節點拖曳動作**。

### 5.2 大腦標記與關聯物件 (V2.0 規劃)
*   **向量貼紙與標記**：內建高品質向量插圖與功能性標記（優先級標籤、進度圖示、情感表情等）。
*   **節點資訊與關聯**：
    *   **關聯線 (Relationship)**：用虛線連接兩個無直接層級關係的節點，並可加上標籤說明。
    *   **外框 (Boundary)**：將多個相關聯的子節點框在一起，並賦予外框標題。
    *   **概要 (Summary)**：為多個選定的分支做一個總結性的子節點。
*   **多媒體與附加檔案**：支援插入超連結、語音備忘錄、本地檔案附件、文字標籤（Label）與詳細備註（Note）。
*   **LaTeX 數學公式**：支援直接輸入 LaTeX 語法，完美渲染數學、物理等學術公式。

## 6. 資料匯入與匯出規格 (V1.0 已實作 & V2.0 規劃)

```mermaid
graph TD
    subgraph 匯入源 (Import)
        Word[Word .docx - V2.0]
        MD[Markdown .md - V1.0 已實作]
        JSON[JSON 檔案 - V1.0 已實作]
        Legacy[舊版 XMind/MindManager - V2.0]
    end

    subgraph MindSprout [MindSprout 編輯器]
        Core[心智圖 / 大綱]
    end

    subgraph 匯出格式 (Export)
        Img[高解析度圖片 PNG - V1.0 已實作 / SVG - V2.0]
        Doc[Word/PPT/Excel - V2.0]
        JSON_E[JSON 檔案 - V1.0 已實作]
        MD_E[Markdown 檔案 - V1.0 已實作]
    end

    Word -.-> Core
    MD --> Core
    JSON --> Core
    Legacy -.-> Core

    Core --> Img
    Core -.-> Doc
    Core --> JSON_E
    Core --> MD_E
```

### 6.1 匯入格式支援
*   **JSON 專案檔 (V1.0 已實作)**：匯入 MindSprout 專屬的心智圖結構檔（具有虛擬根節點相容機制，會自動將舊版的單一根節點心智圖升級為多專案/多自由主題架構）。
*   **Markdown 檔案 (V1.0 已實作)**：將標準 Markdown 縮排列表（Indented List）檔案解析並自動轉換為心智圖。
*   **文書與大綱 (V2.0 規劃)**：Microsoft Word (`.docx`), OPML, TextBundle。
*   **心智圖格式 (V2.0 規劃)**：MindManager (`.mmap`), FreeMind (`.mm`), XMind 官方格式。

### 6.2 匯出格式支援
*   **JSON 專案檔 (V1.0 已實作)**：匯出完整專案資料。
*   **Markdown 檔案 (V1.0 已實作)**：將心智圖遞迴轉換為 Markdown 縮排分級列表。
*   **高解析度 PNG 圖片 (V1.0 已實作)**：透過 `html2canvas` 隱藏縮放比例，以兩倍解析度 (2x Scale) 下載清晰、背景透明的畫質圖片。
*   **向量 SVG 圖片 (V2.0 規劃)**：匯出純向量格式，便於無損放大與列印。
*   **文書與簡報 (V2.0 規劃)**：Microsoft Word (`.docx`), Microsoft PowerPoint (`.pptx`), Microsoft Excel (`.xlsx`), PDF。

## 7. Supabase 資料庫與雲端同步規格 (V1.0 已實作)

*   **資料表結構 (Tables)**：
    *   `public.profiles`：儲存使用者 UID、Email、大頭照頭像與更新時間。
    *   `public.mindmaps`：儲存 UUID、`user_id`（外鍵關聯）、`title` 標題、以及核心 `content`（以 **JSONB** 格式直接保存虛擬根節點及其所有子節點結構，便於未來手機版直接解析同一個 JSON 檔案）。
*   **安全政策 (Row Level Security - RLS)**：
    *   使用者只能存取 `user_id` 與其個人 Auth UID 相同的資料行，確保資料隔離與安全性。
*   **自動化觸發器 (Triggers)**：
    *   在 PostgreSQL 設定 `AFTER INSERT ON auth.users` 觸發器，在新使用者透過 Google OAuth 成功註冊時，自動將頭像與 Email 寫入 `public.profiles`。
